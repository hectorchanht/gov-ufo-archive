# Contributing to realufo.org

Thanks for considering a contribution. This project is a **faithful archive**
of public-domain government records — not a blog, not a fan site, not an
interpretation layer. Most useful PRs land in one of these four buckets:

1. **Add a new national archive** (best ROI — see §1 below).
2. **Fix a dead source URL** in an existing manifest.
3. **Improve an existing case detail page** with verifiable primary-source material.
4. **Tighten the build scripts, CI, or site chrome.**

Project rules are in [CLAUDE.md](CLAUDE.md). Read it first — every PR
follows that spec.

---

## 1. Adding a new national archive

This is the highest-impact contribution. Workflow:

1. **Open a `new-source` issue** describing the agency, source URL, licence,
   and approximate record count. Wait for triage before coding.
2. **Pick a tone colour and seal gradient** (see [CLAUDE.md § 3.1](CLAUDE.md)).
   Use the country's official flag/seal palette where available.
3. **Make `scripts/dl-<slug>.sh`** — idempotent downloader. Direct URL first,
   Wayback fallback if blocked. Model on `scripts/dl-uk.sh` or `dl-chile.sh`.
4. **Make `scripts/build-<slug>.py`** — modelled on `build-nasa.py` (small) or
   `build-aaro.py` (large). Output to `<slug>/index.html`.
5. **Use the shared favicon** at `<slug>/assets/favicon.svg`. Don't ship a
   per-archive favicon — brand recognition matters.
6. **Add nav links** in every other archive's header + footer, plus
   `/about.html`, `/glossary.html`, `/timeline.html`, `/map.html`.
7. **Update `scripts/sync.sh`** to include the new slug in the picker.
8. **Update `scripts/build-api.py` + `build-feeds.py` + `build-og.py` +
   `build-geo.py`** so the new slug surfaces in `/api/`, `/feeds/`, OG cards
   and the map.
9. **Document the source URL + licence in [README.md](README.md)** and add
   the term/agency to [`glossary.html`](glossary.html) if there's a new
   acronym.
10. **Open the PR.** CI must pass (`scripts/check-sources.py` returns 0 on the
    new manifest; `scripts/validate-manifests.py` accepts the shape).

PR checklist (paste into the description):

```
- [ ] dl-<slug>.sh idempotent (cache hit skips, partial download cleaned up)
- [ ] build-<slug>.py uses `git ls-files` (not os.listdir) for a.local
- [ ] Manifest embedded as <script id="arch-data" type="application/json">
- [ ] Page is self-contained (no external build, no JS framework)
- [ ] 360 px viewport tested in browser
- [ ] CSP meta still parses; no new third-party origins introduced
- [ ] sitemap.xml updated
- [ ] /api/all.json regenerates without error
- [ ] /feeds/<slug>.xml regenerates without error
- [ ] Per-archive og.svg generated by scripts/build-og.py
- [ ] CHANGELOG.md updated under [Unreleased]
- [ ] README + glossary updated
```

## 2. Fixing a dead source URL

Run [`scripts/check-sources.py`](scripts/check-sources.py) locally to confirm
the URL is genuinely dead, then either:

- **Update the manifest entry** with the new official URL, OR
- **Add a Wayback fallback** — `https://web.archive.org/web/<ts>id_/<url>`
  with the latest viable snapshot.

Open a tiny PR. Reference the row(s) you changed and the new HEAD-response.

## 3. Improving a case detail page

Acceptable additions:

- Newly declassified primary-source material (FOIA result, NARA accession, etc.).
- Tighter language that reduces editorialising; closer to source-agency phrasing.
- A missing timeline event with a citation.
- Better linked-evidence URLs.

**Not** acceptable:

- Speculative content (ETH claims, "obviously alien", "definitely a hoax", etc.).
- Tabloid-style framing.
- Adding non-government sources as primary evidence.

## 4. Tightening build, CI, site chrome

Welcome. Aim for surgical changes. Don't rewrite the design system without
a prior issue. No build-tool / framework adoption — the site is intentionally
zero-dependency at runtime.

## Local development

```bash
git clone https://github.com/<you>/war-gov-ufo-release
cd war-gov-ufo-release
pip install curl_cffi      # only needed for Akamai-fronted scrapers
./scripts/sync.sh --no-videos   # quick rebuild, no large video downloads
python3 -m http.server 8000     # http://localhost:8000/
```

Test in **Mobile Safari + Chrome Android** at 360 px first. We are
**mobile-first** — desktop polish is a bonus.

## Style

- Code: 2-space JS, 4-space Python, ASCII-art-clean inline CSS.
- Commit messages: imperative present (`add chile favicon`, not `added chile favicon`).
- One concern per PR. Atomic commits beat squash bundles.
- No emojis in commit messages or PR titles. The site uses no emojis either.
