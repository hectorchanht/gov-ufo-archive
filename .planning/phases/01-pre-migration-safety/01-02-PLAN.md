<objective>
Implement PMS-01 — generate and commit `URL-CONTRACT.txt`, the immutable URL contract snapshot taken from `main` BEFORE any SSG code lands. This file is the regression target for Phase 2's CI gate (deferred per D-04) and the source of truth for `_redirects` planning in Phase 2.

Purpose: Per research/PITFALLS.md Pitfall #2, SSG migrations break URL contracts silently. Astro/Eleventy default trailing-slash policies disagree with GitHub Pages defaults; Cloudflare Pages disagrees with both. Without a frozen snapshot from `main` taken NOW, the post-migration site has no defensible "did we drop a URL?" check.

Output: One Python script (`scripts/snapshot-urls.py`, modelled on `sync-nav.py`'s traversal pattern) and one committed text file (`URL-CONTRACT.txt`).
</objective>

  <action>
    Create `scripts/snapshot-urls.py` as a stdlib-only Python script that:

    1. Walks every git-tracked `*.html` file at the repo root and one level deep (root utility pages + each archive's `index.html` + story / case-detail subpages). Use `git -C $REPO ls-files '*.html'` first; if that fails (no git), fall back to `glob.glob` under `REPO/**.html` and `REPO/*/*.html`. Match the `REPO = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))` idiom from `sync-nav.py`.

    2. For each tracked HTML file, derive its public URL using the GitHub Pages convention this repo currently follows (per research/PITFALLS.md §Pitfall 2 + CLAUDE.md §4):
       - `index.html` at repo root → `/`
       - `<slug>/index.html` → `/<slug>/`
       - Any other `<path>/<name>.html` → `/<path>/<name>.html` (preserves the loose-file pattern for `search.html`, `timeline.html`, `aaro/details.html`, `aaro/tic-tac.html`, etc.)
       Convert OS path separators to `/`. Skip files under `.git/`, `node_modules/`, `bundles/`, `assets/vendor/`, `.cache/`, `.pdftext/`, `__pycache__/`.

    3. For each tracked HTML file, also extract per-card identifiers from any `<script id="arch-data" type="application/json">…</script>` OR `<script id="archive-manifest" type="application/json">…</script>` block. Use a non-greedy regex `re.compile(r'<script id="(?:arch-data|archive-manifest)" type="application/json">(.*?)</script>', re.DOTALL)`. Parse the JSON payload (`json.loads(...)`) and emit one URL per asset of the shape `<page-url>#card-<derived-id>`. Derivation rules per the `archive.py` card data schema (`{ t, ti, de, … }`):
       - Prefer an `"id"` field if present on the asset.
       - Otherwise derive a stable slug from `ti` (title): lowercase, replace any run of non-[a-z0-9] with a single `-`, strip leading/trailing `-`, truncate to 80 chars. Per research/PITFALLS.md Pitfall #2 this anchor namespace is what later SRC-04 work depends on.
       - If derivation collides (two assets reduce to the same slug), suffix `-2`, `-3`, … on the second+ occurrence in order of appearance.
       - If the JSON in `<script id="...">` is malformed (rare — present in research notes as a possibility), skip the assets list for that page but still emit the page URL. Print a warning to stderr.

    4. Sort the combined list of page URLs + `#card-<id>` anchor URLs ascending (plain `sorted()`). Deduplicate.

    5. Emit `URL-CONTRACT.txt` with a header block:
       ```
       # URL-CONTRACT.txt
       # Snapshot taken from main @ <short-sha> on <YYYY-MM-DD>
       # Total routes + anchors: <count>
       # Generator: scripts/snapshot-urls.py (idempotent — re-run to refresh)
       ```
       Followed by one URL per line. Use a single trailing newline. Short SHA via `git rev-parse --short HEAD`; date via `datetime.datetime.now(datetime.timezone.utc).strftime('%Y-%m-%d')`.

    6. CLI: `python3 scripts/snapshot-urls.py` writes `URL-CONTRACT.txt` at repo root. Support a `--stdout` flag that prints to stdout instead of writing (useful for diffing). Support `--check` (CI mode) which regenerates in memory and exits non-zero if the file on disk differs — this prepares Phase 2's CI gate (D-04 defers the gate itself, but the script must already be `--check` capable).

    7. Stdlib only — `os`, `sys`, `re`, `json`, `glob`, `subprocess`, `datetime`, `argparse`. No `BeautifulSoup` / `lxml` dependency (per CLAUDE.md §6.2 "stdlib-only except curl_cffi" convention and Claude's-discretion note in 01-CONTEXT.md choosing `re` over BS4).

    8. Module docstring mirrors `sync-nav.py`'s shape: purpose, what it walks, CLI usage, exit codes.

    Banner comment at the top must include: `# Generated input for the Phase 2 URL-CONTRACT drift gate. See .planning/REQUIREMENTS.md PMS-01.`
