<objective>
Execute PMS-03 — a bilateral, 1-day Akamai egress spike that decides Phase 5's scrape architecture. Per `.planning/research/PITFALLS.md` Pitfall #3 and `.planning/research/STACK.md`, the migration assumes moving the scraper from GitHub Actions to Cloudflare Workers cron will help with Akamai bot-blocks on war.gov / aaro.mil. The assumption may be wrong — Cloudflare's edge IPs are flagged in Akamai's IP reputation database, while Azure-hosted Actions runners carry less stigma. Without an objective per-source measurement, Phase 5 cannot rationally choose between a Workers-only, Actions-only, or hybrid architecture.

Purpose: Produce a per-source verdict (D-11) so Phase 5 can split source domains into Workers-friendly and Actions-only buckets, then implement the hybrid scrape lane in SCRP-02 with confidence rather than guessing.

Output: A reproducible spike toolkit under `.planning/spikes/01-akamai/` (Worker code, Actions runner, raw results JSON, README runbook) AND a decision document at `.planning/decisions/akamai-spike.md` — the first ADR in this project, establishing the `.planning/decisions/` folder convention for later cross-phase decisions (D-12).

Sequencing note: This is the Akamai-spike plan. Phase 5 (Scrape Automation) is BLOCKED on the existence of `.planning/decisions/akamai-spike.md` per ROADMAP.md §Cross-Phase Dependencies item 2 + D-08.
