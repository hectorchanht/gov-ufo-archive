<!-- The kill-switch SW contract being installed (per D-05): -->

```javascript
// VERSION stamping is preserved — build-sw.py rewrites this line on every deploy.
// Cache prefix is 'realufo-killswitch-' so it cannot collide with the future Phase 6 SW.
const VERSION = '<git-short-sha>-<YYYYMMDD>';

// install: skipWaiting so the kill-switch takes over immediately
self.addEventListener('install', (event) => {
  self.skipWaiting();
});

// activate: full nuke
self.addEventListener('activate', (event) => {
  event.waitUntil((async () => {
    // (1) unregister self
    await self.registration.unregister();
    // (2) delete every cache (the old shell + data + img caches, plus any orphans)
    const keys = await caches.keys();
    await Promise.all(keys.map((k) => caches.delete(k)));
    // (3) signal all controlled clients to soft-reload
    const clientsList = await self.clients.matchAll({ type: 'window' });
    for (const client of clientsList) {
      client.postMessage({ type: 'sw-killswitch-reload' });
    }
    // (4) claim so future page loads see the no-controller state
    await self.clients.claim();
  })());
});

// fetch: no handler (or a pass-through). Per D-05 the goal is for the SW to
// stop intercepting requests entirely after one reload.

// No message handler is required; the kill-switch is one-shot.
```

<!-- Page-side handler (informational — already lives inline in <head> on root pages; -->
<!-- NOT modified by this plan, but the SW posts to it; documented here so verification understands the round-trip): -->

Pages do not need to add a `message` listener for the kill-switch to function — the unregister + cache clear happens regardless. The postMessage is best-effort UX (gives the page a chance to show a "we've cleaned up your cache" toast). Adding a listener is deferred (would touch every HTML page; out of scope for Phase 1).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer→sw.js | Service worker code runs in user browsers; correctness is load-bearing for offline-first promise. |
| sw.js→browser cache APIs | `caches.delete()` is destructive but idempotent and scoped to the origin's caches only. |
| sw.js→clients.postMessage | Best-effort signal to pages; pages without a listener silently ignore it (acceptable per D-05). |
