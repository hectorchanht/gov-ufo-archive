<objective>
Fix the wargov lightbox that user reported broken on preview deploys (`https://1c266693.realufo.pages.dev`, `https://e0196623.realufo.pages.dev`).

Root cause per 04-RESEARCH.md §7: two bugs.
  (Bug 1) `data-idx` on btn-open is the GLOBAL row index 0..221, but `window.__lbList` is built in DOM order from currently-visible cards — index drift on filter/sort/lazy-load.
  (Bug 2) `btn-open` and `btn-download` both use `href={url}` with no separate `local` propagation, so the lightbox's local-vs-remote PDF branch never reaches the "remote" path → cross-origin PDFs blank-iframe.

Surgical patches A–D from 04-RESEARCH.md §7 — NO refactor.

Purpose: This plan BLOCKS every archive port (Wave 3+) because the same Card.astro / invariants.ts / normalize-csv.py render_card_html() contract is the template that 14 archive ports duplicate. Fixing once benefits all 15 archives.

Output: Patched Card.astro + invariants.ts + normalize-csv.py (byte-equivalent shard HTML preserved per D-10 LOCKED) + new Playwright spec asserting the contract.
</objective>
