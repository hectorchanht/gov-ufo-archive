<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build-time→R2 | rclone sync writes from GH Actions runner to R2 bucket |
| client→R2 | Browser fetches PDFs/videos from assets.realufo.org cross-origin |
| client→GHA secrets | CLOUDFLARE_R2_* secrets only exposed to r2-sync workflow runner |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Information Disclosure | R2 API tokens in workflow logs | mitigate | Use ${{ secrets.* }} interpolation only; never echo secret values; rclone config written to ~/.config/rclone/rclone.conf with file mode 600 |
| T-04-06 | Tampering | Unauthorized R2 writes by leaked token | mitigate | R2 API token scoped to single bucket `realufo` (operator-task in dashboard); token rotation procedure documented in r2-setup.md |
| T-04-07 | Denial of Service | Runaway rclone sync from infinite-loop workflow | mitigate | concurrency.cancel-in-progress: false; timeout-minutes: 30; rclone --checksum makes re-runs idempotent (no duplicate billable ops) |
| T-04-08 | Spoofing | Cross-origin asset substitution via CORS misconfiguration | mitigate | AllowedOrigins explicitly enumerates realufo.org + *.realufo.pages.dev; no wildcard origin; AllowedMethods limited to GET+HEAD (no PUT/DELETE) |
| T-04-SC | Tampering | Supply-chain risk for rclone install script | mitigate | rclone install script (rclone.org/install.sh) is upstream-official; if research/PITFALLS.md package legitimacy gate triggers in future, pin rclone version explicitly via `rclone --version` check |
</threat_model>
