<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `data/*.json` → Astro build | Build-time data ingestion; malformed JSON or schema-incompatible rows are untrusted input to the type-narrowing layer |
| Future scrape pipeline (Phase 5) → `data/<slug>.json` | Workers cron writes will land here; schema gates the inputs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | `data/<slug>.json` mutation that bypasses schema (e.g. type drift in `assets[].t`) | mitigate | Zod schema with `z.enum` on the type field — drift fails build loudly; Phase 5 scrape pipeline will use the same schema for round-trip validation |
| T-03-05 | Information Disclosure | URL fields in `assets[].u` could be tricked into pointing at internal CF Pages preview origins | accept | Phase 3 scope is fixity, not authorization; URLs are public-domain government docs by definition (CLAUDE.md §9) |
| T-03-06 | Denial of Service | A `data/<slug>.json` of unbounded size could slow build to >20min CF Pages cap | accept | wargov is ~261 rows (CSV); other archives in Phase 3 are 0-row skeletons; size budgets are a Phase 4 concern (PERF-01) |
</threat_model>
