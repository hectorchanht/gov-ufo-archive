## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `archiveSlug` prop value → CSS variable + license lookup | Untrusted-string-shaped (props can be anything a page passes); table-driven lookup prevents arbitrary CSS injection |
| Inline `<script is:inline>` content | Trust boundary intentionally accepted — scripts run with same-origin DOM access; mitigation is that they are static, hand-authored, and visible to PR review |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Tampering | A page passes a typo or attacker-controlled `archiveSlug` to RootLayout → CSS-variable injection or license lookup collision | mitigate | TypeScript union type `Props['archiveSlug']: 'wargov'|'aaro'|...` (15-literal union); compile-time check catches typos. Runtime guard: if `archiveSlug` not in TONE map, default to wargov gold AND console.warn (`process.env.NODE_ENV !== 'production'`) |
| T-03-12 | Information Disclosure | OG meta tags leak draft titles before publish | accept | All archive pages are public-domain content; no draft state |
| T-03-13 | XSS | Inline `<script is:inline>` could be exploited if archiveSlug were templated into the script body | mitigate | archiveSlug is NEVER inlined into JS — only CSS variable values (already sanitised by the TONE map) and license string text in Footer body. Scripts are pure DOM-traversal + event-listener attachment. |
| T-03-14 | Tampering | SW registration accidentally points at a different URL | mitigate | Hard-coded literal `'/sw.js'` — no template variable; identical to current main per Phase 1 PMS-02 |
</threat_model>
