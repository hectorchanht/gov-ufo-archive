    CLI:
    `python3 scripts/release-upload.py --slug <slug> --asset-type <pdfs|videos> --scraped-at <ISO> --files <local-path> [<local-path> ...]`

    Behaviour:
    1. Parse `--scraped-at` ISO timestamp; compute quarter via month: month 1–3 = q1, 4–6 = q2, 7–9 = q3, 10–12 = q4. Build tag: `<slug>-<asset-type>-<YYYY>q<N>`.
    2. Verify the tag exists (create if missing): `gh release view <tag>` → if non-zero, `gh release create <tag> --notes "Auto-generated by Plan 05-05 ingest.yml ($scraped_at). Per-archive-quarter scheme per CONTEXT.md D-07." --draft=false`.
    3. For each `--files` entry (process serially — never parallel, per D-08):
       a. Stat the file. If `size > 2_000_000_000` (2 GB literal — D-09 SCRP-10), `print` a structured `R2_OVERFLOW slug=<slug> file=<basename> size=<bytes>` log line, then `continue` (skip the upload — file stays R2-only per D-09).
       b. Compute local SHA-256 via `hashlib.sha256` chunked read (8192-byte blocks).
       c. Check whether the asset already exists on the release. Use `gh release view <tag> --json assets --jq '.assets[].name'`; if `<basename>` is in the list, fetch its `digest` field (GH stores `sha256:...` for assets uploaded with `--digest`) OR run a HEAD against the asset's `browser_download_url` to read `etag` (GH ETag often correlates with SHA but is not guaranteed — fall back to a `gh release download` + local hash if needed for absolute idempotency; the simple path here is the asset-name + size match treated as idempotent).
       d. If asset exists AND remote matches local SHA: `print` `SKIP slug=<slug> file=<basename> reason=sha-match` and `continue`.
       e. If asset exists AND remote does NOT match: `gh release delete-asset <tag> <basename> --yes`. Wait 2s (GitHub delete eventual consistency). Then upload.
       f. Upload: `gh release upload <tag> <local-path>` with retry loop (max 5 attempts; exponential backoff 1s, 2s, 4s, 8s, 16s). NEVER use `--clobber` flag — delete-then-upload is the explicit pattern per D-08.
       g. Log `UPLOADED slug=<slug> file=<basename> tag=<tag> sha=<hex>` on success.

    4. Concurrency lock: at start, open a flock on `/tmp/release-upload-<slug>.lock` (slug-scoped — different slugs can upload in parallel from different workflow runs; same slug serialises). Use `fcntl.flock(fd, fcntl.LOCK_EX | fcntl.LOCK_NB)`; if non-blocking acquire fails, `print` "BUSY slug=<slug> — another invocation holds the lock; retrying in 30s" and retry up to 10 times then `sys.exit(2)`.

    5. Exit codes: 0 success (all files processed, possibly with R2_OVERFLOW skips); 1 transient failure exhausted retries; 2 lock contention; 3 invalid args.

    6. Top docstring (Python module docstring) must reference: SCRP-05 (helper name + idempotency requirement), D-08 (no `--clobber`), D-09 (R2 overflow), CLAUDE.md §5.1 (GH Releases URL pattern).
