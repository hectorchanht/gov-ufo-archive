  <action>
    Create workers/scrape-cron/src/lock.ts implementing two exported async functions plus a typed `LockResult` shape:

    1. `acquireLock(env: Env, runId: string): Promise<LockResult>`
       - Reads KV key `cron-lock` via `env.SCRAPE_STATE.get('cron-lock', 'json')`.
       - If returned value is null or undefined: write a JSON-encoded object `{ runId, acquiredAt: <ISO timestamp> }` to KV under key `cron-lock` with no TTL. Return `{ acquired: true, reason: 'fresh' }`.
       - If returned value's `acquiredAt` parses to a timestamp < 30 minutes ago: return `{ acquired: false, reason: 'held-by-<runId>', heldSince: acquiredAt }` (caller exits silently per D-04).
       - If returned value's `acquiredAt` is ≥ 30 minutes old: emit `console.warn` with message including the prior holder's runId, then OVERWRITE the lock with the current invocation's payload. Return `{ acquired: true, reason: 'stale-recovered', priorHolder: priorRunId }`.

    2. `releaseLock(env: Env, runId: string): Promise<void>`
       - Reads current lock value. If `runId` matches the holder, deletes the key. If it does NOT match (rare — implies another invocation overwrote during work), emit `console.warn` and DO NOT delete (avoid stomping a recovery-acquired lock from another invocation).

    3. Export the `Env` interface with the minimum bindings this module touches: `{ SCRAPE_STATE: KVNamespace }`. Plan 05-04 will extend `Env` with `REALUFO: R2Bucket` and the GH secrets — leave that to 05-04.

    4. Export the `LockResult` discriminated union: `{ acquired: true; reason: 'fresh' | 'stale-recovered'; priorHolder?: string } | { acquired: false; reason: string; heldSince: string }`.

    Constraints:
    - Pure functions; no module-level state.
    - File ≤ 80 lines.
    - No fenced code blocks in this plan — executor authors the TypeScript using the typed signatures above as the contract.
    - The 30-minute staleness window is a constant at the top of the file (`const STALE_LOCK_MS = 30 * 60 * 1000;`) so Plan 05-04 can tune if real-world runs exceed it.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cloudflare cron trigger → Worker scheduled() | First-party CF; no auth surface |
| Public internet → Worker fetch() /run path | Manual trigger; gated by `X-Manual-Trigger` header secret |
| Worker → KV `SCRAPE_STATE` | First-party CF binding via wrangler.toml |
| Worker → R2 `realufo` | First-party CF binding via wrangler.toml (Plan 05-04 actually writes; this plan only binds) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Spoofing | Unauthenticated manual /run trigger | mitigate | `X-Manual-Trigger` header check against `MANUAL_TRIGGER_SECRET`; mismatch returns 404 (not 401) to avoid leaking that the trigger surface exists. Secret set via `wrangler secret put` (never in source). Rotation: re-run `wrangler secret put MANUAL_TRIGGER_SECRET` + redeploy. |
| T-05-03-02 | Denial of Service | Lock-held storm (a stuck Worker permanently blocks cron) | mitigate | 30-min staleness window (`STALE_LOCK_MS`) — if lock acquired > 30 min ago, the next cron acquires anyway and logs a warning. Plan 05-07 smoke verifies this path. |
| T-05-03-03 | Denial of Service | Brute-forcing /run secret | accept | CF's default rate-limiting + Workers Paid CPU caps. If brute-force becomes real, Plan 05-04 can add a per-IP token bucket in KV. Out of scope here. |
| T-05-03-04 | Information Disclosure | `console.log` may leak runIds + scheduledTime to CF Logs | accept | runIds are random UUIDs; no PII; scheduledTime is the cron schedule (public knowledge). CF Logs are operator-only. |
| T-05-03-05 | Tampering | KV cron-lock could be cleared externally | accept | Only operator with wrangler auth + namespace id can write to SCRAPE_STATE. If operator deletes the lock manually, the next cron treats it as fresh — that's the intended escape hatch. |
| T-05-03-SC | Tampering | npm packages wrangler + @cloudflare/workers-types + typescript | mitigate | All three are first-party Cloudflare or Microsoft maintained (no [ASSUMED]/[SUS] — same posture as Plan 05-02 T-05-02-SC). The workspace package.json pins major versions. |
</threat_model>
