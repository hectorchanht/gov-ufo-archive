   # Status defaults to 302 if omitted — D-10 explicitly overrides to 200/301

scripts/build-redirects.py:
  CLI:
    python3 scripts/build-redirects.py            # write _redirects
    python3 scripts/build-redirects.py --stdout   # print, no write
    python3 scripts/build-redirects.py --check    # exit 1 if _redirects is stale vs URL-CONTRACT.txt

  Input: URL-CONTRACT.txt at repo root.
  Output: _redirects at repo root.

Rule emission strategy:
  1. Parse URL-CONTRACT.txt; collect routes (lines starting with `/`); drop `# ...` headers.
  2. Split each entry into (path, fragment) — drop fragment; dedupe paths.
  3. For each canonical path P:
     - Emit `P P 200`  (URL rewrite — keeps URL in browser, doesn't actually redirect on Pages)
     - If P ends with `/` and P != `/`: emit `<P without trailing slash> P 301`  (canonical-with-slash redirect per Specifics §2)
     - If P does NOT end with `/` and P != `/index.html`-like: skip (file-style routes like /foia.html stay as-is)
  4. Stamp header comments: `# Generated by scripts/build-redirects.py from URL-CONTRACT.txt @ <sha> on <date>`, `# DO NOT EDIT — re-run scripts/build-redirects.py to regenerate (D-09, D-11)`.
  5. Sort entries deterministically (alphabetical by source path) so re-runs produce byte-identical files.
